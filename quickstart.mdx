This guide walks you through cloning the repository, configuring your environment, and calling the scheduling engine to produce your first optimized schedule.

**Prerequisites:** Node.js 20.x, PostgreSQL 14 or later, and npm.

<Steps>
  <Step title="Clone the repo and install dependencies">
    ```bash
    git clone https://github.com/JhojanRop/scheduly-api.git
    cd scheduly-api
    npm install
    ```
  </Step>

  <Step title="Configure your environment">
    Create a `.env` file in the project root:

    ```bash
    DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/scheduly_db?schema=public"
    JWT_SECRET="replace-with-a-long-random-secret"
    NODE_ENV="development"
    PORT=3000
    ```

    Replace `USER` and `PASSWORD` with your PostgreSQL credentials. The database `scheduly_db` will be created by the migration in the next step.
  </Step>

  <Step title="Set up the database">
    Run migrations and seed the initial data (day-of-week reference rows):

    ```bash
    npx prisma migrate dev
    npx prisma db seed
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run start:dev
    ```

    The API is now available at `http://localhost:3000/api`. The interactive Swagger UI is at `http://localhost:3000/docs`.
  </Step>

  <Step title="Register a user">
    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "username": "jhojanRop",
        "password": "securepass123",
        "fullName": "Jhojan Ropero"
      }'
    ```

    ```json response
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "username": "jhojanRop",
      "fullName": "Jhojan Ropero",
      "createdAt": "2026-03-15T10:00:00.000Z",
      "updatedAt": "2026-03-15T10:00:00.000Z"
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Log in and get your access token">
    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/auth/login \
      -H "Content-Type: application/json" \
      -c cookies.txt \
      -d '{
        "username": "jhojanRop",
        "password": "securepass123"
      }'
    ```

    ```json response
    {
      "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    }
    ```
    </CodeGroup>

    Save the `accessToken` value. You'll pass it as a Bearer token in every subsequent request. The `-c cookies.txt` flag saves the `refreshToken` HttpOnly cookie automatically.

    Set the token as a shell variable for convenience:

    ```bash
    TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
    ```
  </Step>

  <Step title="Create a subject">
    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/subjects \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Algorithms",
        "code": "CS301"
      }'
    ```

    ```json response
    {
      "id": "sub-uuid-1111",
      "name": "Algorithms",
      "code": "CS301",
      "createdAt": "2026-03-15T10:01:00.000Z"
    }
    ```
    </CodeGroup>

    Note the subject `id` — you'll need it when generating the schedule.
  </Step>

  <Step title="Create a professor">
    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/professors \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Dr. García"
      }'
    ```

    ```json response
    {
      "id": "prof-uuid-2222",
      "name": "Dr. García",
      "createdAt": "2026-03-15T10:02:00.000Z"
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Create a section">
    A section represents a specific time slot and day(s) for a subject, taught by a professor.

    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/sections \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "subjectId": "sub-uuid-1111",
        "professorId": "prof-uuid-2222",
        "startTime": "08:00",
        "endTime": "10:00",
        "dayIds": [1, 3]
      }'
    ```

    ```json response
    {
      "id": "sec-uuid-3333",
      "subjectId": "sub-uuid-1111",
      "professorId": "prof-uuid-2222",
      "startTime": "08:00",
      "endTime": "10:00",
      "days": [
        { "id": 1, "name": "Monday" },
        { "id": 3, "name": "Wednesday" }
      ]
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Create a rule">
    Rules tell the engine what you prefer. Add as many as you like with different priorities.

    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/rules \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "type": "NO_EARLY_MORNINGS",
        "priority": 5
      }'
    ```

    ```json response
    {
      "id": "rule-uuid-4444",
      "type": "NO_EARLY_MORNINGS",
      "priority": 5,
      "params": null,
      "createdAt": "2026-03-15T10:04:00.000Z"
    }
    ```
    </CodeGroup>

    <Tip>
      Higher `priority` values carry more weight in scoring. Use a consistent scale (e.g. 1–10) across all your rules so the relative importance is clear.
    </Tip>
  </Step>

  <Step title="Generate your schedule">
    Pass the IDs of the subjects you want scheduled. The engine returns the top three conflict-free combinations ranked by score.

    <CodeGroup>
    ```bash curl
    curl -X POST http://localhost:3000/api/scheduling/generate \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "subjectIds": ["sub-uuid-1111"]
      }'
    ```

    ```json response
    [
      {
        "score": 42,
        "sections": [
          {
            "id": "sec-uuid-3333",
            "subjectId": "sub-uuid-1111",
            "professorId": "prof-uuid-2222",
            "startTime": "08:00",
            "endTime": "10:00",
            "days": [
              { "id": 1, "name": "Monday" },
              { "id": 3, "name": "Wednesday" }
            ]
          }
        ]
      }
    ]
    ```
    </CodeGroup>

    The response is an array of up to three schedule objects, sorted from highest to lowest score. Pick the one you prefer and save it with `POST /api/saved-schedules`.
  </Step>
</Steps>
