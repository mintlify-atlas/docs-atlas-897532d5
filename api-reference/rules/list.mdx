---
title: "GET /api/rules"
description: "Retrieve all rules belonging to the authenticated user."
---

## Overview

Returns every rule created by the authenticated user. Rules are ordered by `priorityOrder` ascending, so the first element in the array is the highest-priority rule applied by the scheduling engine.

Rules from other users are never included in the response.

## Request

### HTTP method and path

`GET /api/rules`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Request body

None.

## Response

HTTP `200 OK`

An array of rule objects ordered by `priorityOrder` ascending. Returns an empty array `[]` when you have no rules.

<ResponseField name="[]" type="object[]" required>
  <Expandable title="rule properties">
    <ResponseField name="id" type="string" required>
      UUID that uniquely identifies the rule.
    </ResponseField>
    <ResponseField name="type" type="string" required>
      Identifier for the rule type (e.g., `NO_GAPS`, `LUNCH_BREAK_PROTECTED`).
    </ResponseField>
    <ResponseField name="priorityOrder" type="number" required>
      Evaluation order. Lower values are applied first.
    </ResponseField>
    <ResponseField name="parameters" type="object | null" required>
      Rule-specific configuration, or `null` if the rule requires no parameters.
    </ResponseField>
    <ResponseField name="userId" type="string" required>
      UUID of the user who owns this rule.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
