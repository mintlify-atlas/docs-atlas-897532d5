---
title: "GET /api/sections/:id"
description: "Retrieve a single section by its ID."
---

## Overview

Returns a single section by UUID, including its subject, professor, and associated days. The section must belong to a subject owned by the authenticated user — attempting to access a section whose subject belongs to another user returns `404`.

<Note>
  Day IDs use a fixed reference: 1 = Monday, 2 = Tuesday, 3 = Wednesday, 4 = Thursday, 5 = Friday, 6 = Saturday, 7 = Sunday.
</Note>

## Request

### HTTP method and path

`GET /api/sections/:id`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Path parameters

<ParamField path="id" type="string" required placeholder="d4e5f6a7-b8c9-0d1e-2f3a-4b5c6d7e8f9a">
  UUID of the section to retrieve.
</ParamField>

## Response

HTTP `200 OK`

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the section.
</ResponseField>

<ResponseField name="startTime" type="string" required>
  Start time as an ISO 8601 timestamp.
</ResponseField>

<ResponseField name="endTime" type="string" required>
  End time as an ISO 8601 timestamp.
</ResponseField>

<ResponseField name="subjectId" type="string" required>
  UUID of the parent subject.
</ResponseField>

<ResponseField name="professorId" type="string">
  UUID of the assigned professor, or `null` if unassigned.
</ResponseField>

<ResponseField name="sectionDays" type="object[]" required>
  <Expandable title="sectionDay properties">
    <ResponseField name="id" type="string" required>
      UUID of the section-day join record.
    </ResponseField>
    <ResponseField name="sectionId" type="string" required>
      UUID of the parent section.
    </ResponseField>
    <ResponseField name="dayId" type="number" required>
      Day ID (1–7).
    </ResponseField>
    <ResponseField name="day" type="object" required>
      <Expandable title="day properties">
        <ResponseField name="id" type="number" required>
          Numeric day ID.
        </ResponseField>
        <ResponseField name="name" type="string" required>
          Lowercase day name (e.g., `"thursday"`).
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
| `404 Not Found` | No section with the given ID exists for the authenticated user. This is returned both when the ID does not exist and when it belongs to a different user's subject. |
