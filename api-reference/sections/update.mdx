---
title: "PUT /api/sections/:id"
description: "Replace all fields of an existing section."
---

## Overview

Updates a section by replacing its time slot, days, and professor assignment. The section must belong to a subject owned by the authenticated user. All existing `sectionDays` records are deleted and recreated from the `dayIds` provided in the request.

<Note>
  Day IDs use a fixed reference: 1 = Monday, 2 = Tuesday, 3 = Wednesday, 4 = Thursday, 5 = Friday, 6 = Saturday, 7 = Sunday.
</Note>

## Request

### HTTP method and path

`PUT /api/sections/:id`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Path parameters

<ParamField path="id" type="string" required placeholder="d4e5f6a7-b8c9-0d1e-2f3a-4b5c6d7e8f9a">
  UUID of the section to update.
</ParamField>

### Request body

<ParamField body="subjectId" type="string" required placeholder="b1c2d3e4-5f6a-7b8c-9d0e-1f2a3b4c5d6e">
  UUID of the subject this section belongs to. The subject must be owned by the authenticated user.
</ParamField>

<ParamField body="startTime" type="string" required placeholder="09:00">
  New start time in `HH:MM` 24-hour format.
</ParamField>

<ParamField body="endTime" type="string" required placeholder="11:00">
  New end time in `HH:MM` 24-hour format.
</ParamField>

<ParamField body="dayIds" type="number[]" required placeholder="[2, 4]">
  Array of day IDs indicating which days the section meets. Replaces all previously set days. Each value must be an integer between 1 and 7 inclusive.
</ParamField>

<ParamField body="professorId" type="string" placeholder="e5f6a7b8-9c0d-1e2f-3a4b-5c6d7e8f9a0b">
  UUID of the professor to assign, or omit/`null` to remove the current assignment.
</ParamField>

## Response

HTTP `200 OK`

<ResponseField name="id" type="string" required>
  UUID of the section.
</ResponseField>

<ResponseField name="startTime" type="string" required>
  Updated start time as an ISO 8601 timestamp.
</ResponseField>

<ResponseField name="endTime" type="string" required>
  Updated end time as an ISO 8601 timestamp.
</ResponseField>

<ResponseField name="subjectId" type="string" required>
  UUID of the parent subject.
</ResponseField>

<ResponseField name="professorId" type="string">
  UUID of the assigned professor, or `null` if unassigned.
</ResponseField>

<ResponseField name="sectionDays" type="object[]" required>
  <Expandable title="sectionDay properties">
    <ResponseField name="id" type="string" required>
      UUID of the section-day join record.
    </ResponseField>
    <ResponseField name="sectionId" type="string" required>
      UUID of the parent section.
    </ResponseField>
    <ResponseField name="dayId" type="number" required>
      Day ID (1–7).
    </ResponseField>
    <ResponseField name="day" type="object" required>
      <Expandable title="day properties">
        <ResponseField name="id" type="number" required>
          Numeric day ID.
        </ResponseField>
        <ResponseField name="name" type="string" required>
          Lowercase day name.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | One or more fields are missing, incorrectly formatted, or contain out-of-range day IDs. |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
| `404 Not Found` | No section, subject, or professor with the given ID exists for the authenticated user. |
