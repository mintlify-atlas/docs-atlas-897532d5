---
title: "PUT /api/saved-schedules/:id"
description: "Rename an existing saved schedule."
---

## Overview

Updates the display name of a saved schedule. This is the only field you can change after creation — section membership is fixed at the time of creation. The schedule must belong to the authenticated user.

<Note>
  To change which sections are included in a saved schedule, delete the existing one and create a new one with the desired `sectionIds`.
</Note>

## Request

### HTTP method and path

`PUT /api/saved-schedules/:id`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Path parameters

<ParamField path="id" type="string" required placeholder="c3d4e5f6-a7b8-9c0d-1e2f-a3b4c5d6e7f8">
  UUID of the saved schedule to update.
</ParamField>

### Request body

<ParamField body="name" type="string" required placeholder="My updated schedule name">
  New display name for the saved schedule. Maximum 50 characters.
</ParamField>

## Response

HTTP `200 OK`

<ResponseField name="id" type="string" required>
  UUID of the saved schedule.
</ResponseField>

<ResponseField name="name" type="string" required>
  The updated display name.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp indicating when the schedule was originally saved. This value does not change on update.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this saved schedule.
</ResponseField>

<ResponseField name="schedulesSections" type="object[]" required>
  The existing junction records and section details, unchanged.
</ResponseField>

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | `name` is missing or exceeds 50 characters. |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
| `404 Not Found` | No saved schedule with the given ID exists for the authenticated user. |
