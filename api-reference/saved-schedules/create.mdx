---
title: "POST /api/saved-schedules"
description: "Persist a generated schedule by saving its section IDs."
---

## Overview

Saves a schedule generated by `POST /api/scheduling/generate`. You provide a display name and the list of `sectionId` values from the scheduling result. The API creates junction records linking the saved schedule to each section.

<Info>
  To obtain the `sectionIds`, first call `POST /api/scheduling/generate` and copy the `sectionId` from each entry in the `sections` array of the result you want to save.
</Info>

## Request

### HTTP method and path

`POST /api/saved-schedules`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Request body

<ParamField body="name" type="string" required placeholder="My ideal schedule">
  Display name for the saved schedule. Maximum 50 characters.
</ParamField>

<ParamField body="sectionIds" type="string[]" required>
  An array of section UUIDs to include in this saved schedule. Must contain at least one element. Each value must be a valid v4 UUID corresponding to a section returned by the scheduling engine.
</ParamField>

## Response

HTTP `201 Created`

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the saved schedule.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the saved schedule.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp indicating when the schedule was saved.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this saved schedule.
</ResponseField>

<ResponseField name="schedulesSections" type="object[]" required>
  Junction records linking this schedule to its sections.
  <Expandable title="section entry properties">
    <ResponseField name="scheduleId" type="string" required>
      UUID of the saved schedule.
    </ResponseField>
    <ResponseField name="sectionId" type="string" required>
      UUID of the linked section.
    </ResponseField>
    <ResponseField name="section" type="object" required>
      Full section details.
      <Expandable title="section properties">
        <ResponseField name="id" type="string" required>
          UUID of the section.
        </ResponseField>
        <ResponseField name="startTime" type="string" required>
          Section start time (ISO 8601 time with timezone).
        </ResponseField>
        <ResponseField name="endTime" type="string" required>
          Section end time (ISO 8601 time with timezone).
        </ResponseField>
        <ResponseField name="subjectId" type="string" required>
          UUID of the subject this section belongs to.
        </ResponseField>
        <ResponseField name="professorId" type="string | null" required>
          UUID of the assigned professor, or `null`.
        </ResponseField>
        <ResponseField name="subject" type="object" required>
          The subject linked to this section.
        </ResponseField>
        <ResponseField name="professor" type="object | null" required>
          The professor linked to this section, or `null`.
        </ResponseField>
        <ResponseField name="sectionDays" type="object[]" required>
          Days of the week when this section meets.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

    "name": "My ideal schedule",

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | `name` exceeds 50 characters, `sectionIds` is empty, or a value in `sectionIds` is not a valid UUID. |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
