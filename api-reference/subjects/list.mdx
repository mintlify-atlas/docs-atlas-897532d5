---
title: "GET /api/subjects"
description: "Retrieve all subjects belonging to the authenticated user."
---

## Overview

Returns every subject that was created by the authenticated user. Subjects from other users are never included in the response.

## Request

### HTTP method and path

`GET /api/subjects`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Request body

None.

## Response

HTTP `200 OK`

An array of subject objects. Returns an empty array `[]` when the user has no subjects.

<ResponseField name="[]" type="object[]" required>
  <Expandable title="subject properties">
    <ResponseField name="id" type="string" required>
      UUID that uniquely identifies the subject.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Display name of the subject.
    </ResponseField>
    <ResponseField name="userId" type="string" required>
      UUID of the user who owns this subject.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

curl --request GET \

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
