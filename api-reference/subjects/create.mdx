---
title: "POST /api/subjects"
description: "Create a new subject belonging to the authenticated user."
---

## Overview

Creates a subject and associates it with the authenticated user. Subjects are used to group class sections (e.g., "Mathematics", "Physics"). Each user manages their own independent list of subjects.

## Request

### HTTP method and path

`POST /api/subjects`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Request body

<ParamField body="name" type="string" required placeholder="Mathematics">
  Display name for the subject. Maximum 100 characters.
</ParamField>

## Response

HTTP `201 Created`

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the subject.
</ResponseField>

<ResponseField name="name" type="string" required>
  The name of the subject as provided in the request.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this subject.
</ResponseField>

## Example

<CodeGroup>

```bash curl
curl --request POST \
  --url https://api.scheduly.app/api/subjects \
  --header 'Authorization: Bearer <accessToken>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Mathematics"
  }'
```

```json Response
{
  "id": "b1c2d3e4-5f6a-7b8c-9d0e-1f2a3b4c5d6e",
  "name": "Mathematics",
  "userId": "a3f8c2d1-4b5e-4f6a-8c9d-0e1f2a3b4c5d"
}
```

</CodeGroup>

## Error responses

| Status | Description |
| ------ | ----------- |
| `400 Bad Request` | The `name` field is missing or exceeds 100 characters. |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
