---
title: "GET /api/subjects/:id"
description: "Retrieve a single subject by its ID."
---

## Overview

Returns a single subject by UUID. The subject must belong to the authenticated user — attempting to access a subject owned by another user returns `404`, the same as requesting a subject that does not exist.

## Request

### HTTP method and path

`GET /api/subjects/:id`

### Authentication

Bearer token required.

```
Authorization: Bearer <accessToken>
```

### Path parameters

<ParamField path="id" type="string" required placeholder="b1c2d3e4-5f6a-7b8c-9d0e-1f2a3b4c5d6e">
  UUID of the subject to retrieve.
</ParamField>

## Response

HTTP `200 OK`

<ResponseField name="id" type="string" required>
  UUID that uniquely identifies the subject.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the subject.
</ResponseField>

<ResponseField name="userId" type="string" required>
  UUID of the user who owns this subject.
</ResponseField>

## Example

<CodeGroup>

```bash curl
curl --request GET \
  --url https://api.scheduly.app/api/subjects/b1c2d3e4-5f6a-7b8c-9d0e-1f2a3b4c5d6e \
  --header 'Authorization: Bearer <accessToken>'
```

```json Response
{
  "id": "b1c2d3e4-5f6a-7b8c-9d0e-1f2a3b4c5d6e",
  "name": "Mathematics",
  "userId": "a3f8c2d1-4b5e-4f6a-8c9d-0e1f2a3b4c5d"
}
```

</CodeGroup>

## Error responses

| Status | Description |
| ------ | ----------- |
| `401 Unauthorized` | The access token is missing, malformed, or expired. Obtain a fresh token from `/api/auth/refresh`. |
| `404 Not Found` | No subject with the given ID exists for the authenticated user. This is returned both when the ID does not exist and when it belongs to a different user. |
